# 模型、权威来源与热度评分

## 模型选择

在 `.env` 中设置：

```env
OPENAI_API_KEY=你的密钥
OPENAI_MODEL=gpt-5.6
```

如果你的 API 项目提供 `gpt-5.5`，直接将模型名改为：

```env
OPENAI_MODEL=gpt-5.5
```

应用会把 `OPENAI_MODEL` 原样传给 OpenAI Responses API，并在服务启动时验证当前 API 项目是否可访问该模型。若验证失败，报错会包含模型名称并提示检查 API 项目权限或改用可用模型；未配置 API Key 时，AI 工作台保持使用本地规则分析。

## 平台权重

复制 `.env.example` 为 `.env` 后，可以调整以下值：

```env
SOURCE_WEIGHT_ARXIV=1.40
SOURCE_WEIGHT_GITHUB=1.20
SOURCE_WEIGHT_HACKER_NEWS=0.85
SOURCE_WEIGHT_RSS=1.00
SOURCE_WEIGHT_X=1.10
SOURCE_WEIGHT_BILIBILI=1.00
SOURCE_WEIGHT_DEFAULT=1.00
HEAT_FRESHNESS_WEIGHT=1.00
HEAT_ENGAGEMENT_WEIGHT=1.00
```

- 数值越高，同一条内容对话题热度的贡献越高；设为 `0` 时该平台内容仍可展示为证据，但不增加热度。
- 热度由“加权来源数量 + 时效 + 互动指标”构成，最高为 100 分。
- 修改权重后，在页面点击“生成热点”或重启服务后重新聚合，已存在的证据权重也会同步刷新。

## 权威作者白名单

权威性只来自你明确配置的白名单，不根据粉丝数或模型猜测。

```env
# 白名单作者额外加成：0.65 表示在平台基础权重上增加 65%。
AUTHORITY_AUTHOR_BONUS=0.65

# 任意平台按显示作者名匹配，可用于 RSS、GitHub 等。
TRUSTED_AUTHOR_NAMES=OpenAI,Anthropic,Google DeepMind

# X 用户名，逗号分隔；启用采集还必须填写 X_BEARER_TOKEN。
X_BEARER_TOKEN=
X_AUTHOR_USERNAMES=openai,GoogleDeepMind

# B 站作者 MID，逗号分隔。
BILIBILI_AUTHOR_MIDS=12345678,87654321
```

### X

配置 `X_BEARER_TOKEN` 和 `X_AUTHOR_USERNAMES` 后，系统通过官方 X API 拉取白名单作者的公开原创帖文。未配置二者之一时不会请求 X。

### B 站

配置 `BILIBILI_AUTHOR_MIDS` 后，系统仅请求公开作者主页并读取页面直接提供的视频标题、简介、发布时间和公开互动数据。不会绕过登录、验证码、风控、robots 规则或付费限制；页面结构发生变化时，该来源会在采集状态中显示失败，而不会影响其他来源。

## 热点详情

话题详情页新增以下可解释信息：

- 平台覆盖和各平台证据数量；
- 权威作者证据数量；
- 加权来源数量及平均来源权重；
- 每条证据的作者、发布时间、抓取时间、原始链接和关联权重。
